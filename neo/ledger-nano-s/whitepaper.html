Introduction To Cryptocoins and Hardware Wallets

NEO is a cryptocoin that uses 256 bit addresses (called private keys) to govern the transfer of NEO, GAS and other tokens on the NEO blockchain, by digitally signing the transaction that executes the transfer.

As with other cryptocoins, this means anyone that knows this 256 bit address can transfer (and thus sell) the NEO at the given address. For this reason, the 256 bit address must be kept in a safe place so it cannot easily be stolen. At the same time, the address must be accessible so a user can use it to sign transactions.

This can be done by using a piece of hardware to store the private key, and have that same hardware use the key to digitally sign transactions.

Introduction To The Ledger Nano S

The Ledger Nano S is a device that connects to a computer's USB port and can accept data requests via USB. It has a monochrome screen with a small screen resolution, and two buttons at the top.

The idea is you store your private key on the hardware wallet, and then you send transactions to the hardware wallet for signing. You select 'yes' or 'no' by pressing one of the two buttons on the top.

Using The Ledger Nano S with NEO

There are two parts that need to be built in order to use NEO with the hardware wallet:

  1. The software to run on the hardware wallet itself, to sign transactions.
  2. The software to run on the host computer, which will:
    1. create the transaction
    2. send it to the hardware wallet for signing
    3. accept the wallet's decision to sign or not
    4. connect to the NEO network and send the signed transactions

hardware wallet software

    The user can scroll through the below actions using the buttons, and perform an action by clicking both buttons together.

    1. Before any action has been taken, a user can exit using the left hand X. The message displayed is "Wake Up, NEO..."
    2. After a transaction has been sent to the ledger to be signed, a user can navigate down using the left arrow, and up using the right arrow. clicking both buttons performs the action in brackets.
    3. Confirm a transaction and sign, without looking at the details [sign]
    4. View each output asset id, amount, and destination address [no action]
    5. View the total system fee and gas fee. [no action]
    6. Reject Transaction and do not sign. [reject]
    7. Confirm Transaction and sign. [sign]
    8. Exit
